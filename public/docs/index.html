DocWise AI API

Extract clean, structured JSON from Indian GST invoices, purchase orders, and business documents. Built for accounting and ERP software integrators.

Overview

DocWise AI turns scanned invoices and PDFs into machine-readable JSON — parties, GSTINs, line items, taxes, totals — with per-field confidence and GSTIN validation. If you build accounting or ERP software and want to skip building document-parsing yourself, this API drops in as a single HTTP call.

  • Multi-page PDFs are supported — each page is extracted independently to prevent cross-page contamination.
  • Deterministic prompt design and a second-pass verification stage make repeated runs consistent.
  • Response includes overall confidence and warnings so you can auto-approve high-confidence extractions and queue the rest for human review.

Authentication

Every request must include a bearer API key:

Authorization: Bearer YOUR_API_KEY

API keys look like dx_live_<64 hex>. To get a key, contact support — self-serve signup is not open yet.

Missing key returns 401 Missing Bearer api_key. Invalid key returns 401 Invalid api_key. Disabled tenants return 403 Tenant is disabled.

POST /api/v1/extract

Extract structured data from one or more document images or PDFs.

Request — two supported formats

1. JSON body (base64)

Content-Type: application/json

{
  "images": ["<base64>", "<base64>"],   // 1–8 items; PDFs or images
  "hint":   "optional context, max 500 chars"
}

Each item can be raw base64 or a full data:... URI. MIME type is auto-detected from the file's magic bytes (PNG, JPEG, WEBP, GIF, PDF).

2. Multipart form-data (file upload)

Content-Type: multipart/form-data

images: <file>      // repeatable field, up to 8 files
hint:   <string>    // optional

Allowed types: image/jpeg, image/png, image/webp, image/gif, application/pdf. Per-file cap: 4 MB. A single field image is accepted as a convenience alias.

Response — 200 OK

{
  "ok": true,
  "extraction_id": "uuid",
  "created_at": "2026-07-10T12:00:00Z",
  "document_type": "tax_invoice",
  "overall_confidence": 0.92,
  "billed_pages": 3,
  "data": {
    "documents": [
      {
        "document_type": "tax_invoice",
        "overall_confidence": 0.92,
        "invoice_number": "INV/2026/00123",
        "invoice_date": "2026-06-14",
        "seller":   { "name": "...", "gstin": "27ABCDE1234F1Z5", "address": "..." },
        "buyer":    { "name": "...", "gstin": "29XYZAB5678K1Z2", "address": "..." },
        "line_items": [
          { "description": "...", "hsn_sac": "8471", "quantity": 2, "unit_price": 15000, "amount": 30000 }
        ],
        "taxes":   { "cgst": 2700, "sgst": 2700, "igst": 0, "total_tax": 5400 },
        "totals":  { "subtotal": 30000, "total_amount": 35400, "amount_in_words": "..." },
        "warnings": ["gstin_checksum_mismatch"],
        "confidence": { "invoice_number": 0.98, "seller.gstin": 0.99, "...": 0.0 }
      }
    ]
  }
}
Billing note. PDFs are billed per real page count (a 3-page PDF = 3 pages). Images are billed as 1 page each. The billed_pages field in the response reflects what was billed for this call.

GET /api/v1/usage/current

Returns the current (not-yet-invoiced) billing month's usage for the authenticated tenant.

Response — 200 OK

{
  "billing_month": "2026-07-01",
  "total_pages": 1247,
  "total_amount": 1247.00,
  "rate_per_page": 1.00,
  "currency": "INR"
}

GET /api/v1/invoices

Returns the tenant's past invoices, most recent first.

Response — 200 OK

{
  "invoices": [
    {
      "id": "uuid",
      "billing_month": "2026-06-01",
      "total_pages": 4890,
      "total_amount": 4890.00,
      "status": "pending",           // pending | sent | paid | overdue
      "due_date": "2026-07-16",
      "generated_at": "2026-07-01T00:15:00Z",
      "paid_at": null,
      "payment_reference": null
    }
  ],
  "currency": "INR"
}

Billing

  • Model: per-page postpaid, billed monthly in arrears. No usage cap.
  • Rate: your rate_per_page is configured on your tenant account (default ₹1.00/page). Every image counts as 1 page; PDFs count as their real page count.
  • Invoice generation: automated on the 1st of each month at 00:15 UTC for the previous month's usage.
  • Due date: 15 days after invoice generation.
  • Payment: collected offline via bank transfer or cheque against the monthly invoice — no cards, UPI, or gateways required on your side. Invoices are available via GET /api/v1/invoices and are also sent manually.

Code samples

POST /api/v1/extract

curl -X POST https://extract-aha-moment.lovable.app/api/v1/extract \
  -H "Authorization: Bearer $DOCWISE_KEY" \
  -F "images=@invoice.pdf"
import base64, requests

with open("invoice.pdf", "rb") as f:
    b64 = base64.b64encode(f.read()).decode()

r = requests.post(
    "https://extract-aha-moment.lovable.app/api/v1/extract",
    headers={"Authorization": f"Bearer {DOCWISE_KEY}"},
    json={"images": [b64]},
    timeout=120,
)
r.raise_for_status()
print(r.json())
import fs from "node:fs/promises";

const b64 = (await fs.readFile("invoice.pdf")).toString("base64");
const res = await fetch("https://extract-aha-moment.lovable.app/api/v1/extract", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${process.env.DOCWISE_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ images: [b64] }),
});
console.log(await res.json());
<?php
$b64 = base64_encode(file_get_contents('invoice.pdf'));

$ch = curl_init('https://extract-aha-moment.lovable.app/api/v1/extract');
curl_setopt_array($ch, [
  CURLOPT_RETURNTRANSFER => true,
  CURLOPT_POST => true,
  CURLOPT_HTTPHEADER => [
    'Authorization: Bearer ' . getenv('DOCWISE_KEY'),
    'Content-Type: application/json',
  ],
  CURLOPT_POSTFIELDS => json_encode(['images' => [$b64]]),
]);
$response = curl_exec($ch);
echo $response;
